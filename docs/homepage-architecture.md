# 首页结构说明

## 文件职责

- `index.html`：只保留首页结构与资源加载顺序。
- `styles/home.css`：首页视觉、响应式规则和主题样式。
- `scripts/home-layout.js`：背景、门和标题尺寸的纯计算，不操作 DOM。
- `scripts/home.js`：页面数据渲染、交互、动画和 DOM 更新。
- `tests/home-layout.test.cjs`：首页几何计算的多尺寸回归测试。

## 首屏计算顺序

1. 读取视口尺寸和背景 `object-position`。
2. `home-layout.js` 判断使用普通铺满模式还是窄高屏顶部流式模式。
3. 根据同一个缩放比例计算背景和门的位置。
4. 使用“导航右边界到门左边界”的空间计算标题字号。
5. 根据背景下边界定位操作按钮和底部提示。
6. 一次性把计算结果写入 DOM 和画布。

所有尺寸变化都通过 `scheduleLayout()` 合并到下一帧执行，避免一次窗口变化触发多轮重复计算。

## 关键配置

背景原始尺寸、水平面和门的原始坐标位于：

```text
scripts/home-layout.js
```

侧栏与窄条导航宽度位于：

```text
styles/home.css
--sidebar-width
--rail-width
```

更换头图后，只需重新测量并修改 `SOURCE`、`DOOR` 和 `DOOR_PAD`，不要在其他函数中单独补位置偏移。

## 响应式原则

- 宽屏：背景铺满首屏，门、标题和导航共用同一坐标体系。
- 窄高屏：若门在铺满模式下超过画面宽度三分之一，则整张背景一起缩放，门宽控制为画面四分之一。
- 窄高屏：宽度不超过 `760px` 且高宽比达到 `1.35` 时，切换为上下叠放；文字使用导航右侧完整宽度，门位于文字下方并在内容区域中部略偏右。
- 首屏底层：独立水彩底层始终覆盖整个 `.hero`，保证脚本尚未完成定位或前景图需要下移时，顶部和底部都不会露出纯色背景。
- 展开侧栏：桌面端把侧栏宽度计入可用空间；窄屏使用覆盖模式，不挤压首屏。
- 标题：优先保持两条指定文字各自单行，再根据可用空间连续缩小。

## 验证

运行：

```powershell
node tests/home-layout.test.cjs
node --check scripts/home-layout.js
node --check scripts/home.js
```

视觉效果仍需人工确认；自动检查只保证坐标关系、尺寸边界和脚本结构。
