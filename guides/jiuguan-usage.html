02 导入预设
1. 预设是做什么的

预设的功能很丰富，主要用来给 AI 洗脑，引导它的回复。破甲指破除 AI 的道德限制，更自由地展开剧情。

如果你的 AI 总是在重复，可以尝试换一个预设玩一会。

2. 怎么正确导入预设

先要点击第二个【插头】图标，在 API 模式选择【聊天补全】！！！

再进入第一个图标在 【对话补全预设】 后方导入你下载的预设！！！

导入时会弹出一个框，可以拒绝。

3. 预设的正则

下载预设时还要注意这个预设有没有同步发布的正则（如何导入见 06 插件 — 正则部分），比如下图：

4. 预设的使用

使用预设的时候，一定要仔细看楼主的介绍，如果使用不当可能会破甲失败。

预设中有很多具体的条目，可以在下拉菜单中自由打开和关闭你需要的选项（主要是增添文风参考）。

5. 我在使用的预设推荐

这个推荐其实非常有时效性，还是建议大家多去旅程/类脑预设区自己看看，模型和预设迭代都很快。

最近我爱用 Gemini 预设 小猫之神，Claude 很久没玩过了但鹿鹿的印象不错。

03 API 连接
1. 获取 API

先用最简单的 DeepSeek 做示范（暂时搞角色扮演还是差点意思，但入手简单），开开胃，Claude / Gemini 可以作为进阶。

1. DeepSeek 官方（经典好获取，不用梯）

夜间半价，新 API 用户好像有限时的免费额度，不高。

登录官网，进入【API 开放平台】，在左侧的【API keys】中【创建 API keys】，把 key 保存好，这就是你的 API 密钥。

2. Gemini（找公益站是免费）

Gemini 现在的 3.1 我还是很满意的，一般自己搭建 glic 反代，或者蹭公益站。

纯新手找公益站吧，不用花钱，有免费源。虽然 260325 大砍额度，但万一呢。

3. Claude（很贵，但也能吃到）

推荐 Claude Opus 系列模型，4.6 Sonnet 给我的感觉还不如 Gemini 3.0 Pro，但 4 Opus 就很聪明。

现在 PRO 账户想爽玩额度不够，量够的 MAX 一个月太贵了，不推荐。

但可以搞谷歌账号，小号申请加入 PRO 家庭组，申请反重力资格，配置反代，这样可以 Claude Opus，一个号通常活不到一天，但花销也仅限于此，号死了就用小号退家庭组。

2. 填写 API

进入上方工具栏第二项的小插头：

1. DeepSeek —— 官方

【聊天补全】-【DeepSeek】来源 -【填写密钥】- 点击【连接】- 选取模型【deepSeek-chat】

左侧确认好预设，测试成功后，可以顺手保存。

【聊天补全】-【OpenRouter】来源 -【填写密钥】- 点击【连接】- 选取模型【DeepSeek: DeepSeek V3 0324 (free)】，确认好预设后可以另存。

2. Gemini

这次没自己搭反代，用的公益站~

3. Claude

Claude 官方 API 太贵，官 PRO 玩不了口碑好的 Opus 系列模型，所以 ClwdR 反代也走不太通了。

现在可以试试谷歌账号申请反重力资格，然后反代。

3. 当前这几种 API 使用体验的简单对比

DeepSeek 的整体扮演能力弱，但赢在获取和使用简单，听说 V4 会很不错；

Gemini 3.1 Pro 不绝望，情绪丰富，可以各种恨海情天搞 DRAMA 剧情，2.5 Flash 速度很快；

Claude 4.6 Sonnet 不如 Gemini（个人感觉），但 4 Opus 着实很难吃到，搞反重力反代吧。

04 角色卡与游玩

终于到重头戏了，角！色！卡！

1. 查看你的角色

此处可以查看你的全部角色，图示为默认角色。

2. 导入新角色卡与互动

选中角色就可以开始互动了，蓝框处可以导入新角色卡。

3. 其他功能

① 绿框中的地球为角色自带的【世界书】，如果误删了，点这里重导。地球后方，可以为当前角色再绑定一本【世界书】。骷髅头是删除角色。

② 点击角色头像，可以直接更换头像。

③ 可以给角色卡增添标签方便管理。

④ 角色描述可以自行删减，改点设定。

⑤ 第一条消息会是引导故事的重要开局，你甚至可以把设定丢给 AI，让它写几版开场白加进来。在【其他开场/额外问候语】中可以添加不同开场，在对话窗中的箭头处即可切换。

4. 含有世界书/局部正则的角色卡

作为新手，我们不用太顾虑这些方面的问题。因为在我们下载的 PNG 格式的角色卡，在导入时自带这些内容。

当导入角色卡后，作者又更新了新世界书/局部正则，这时候也只要手动添加就行：

1. 在更多中选择替换更新能替换当前角色卡的本体（即作者的最近更新）。

2. 角色卡的正则放在局部正则中，更新时注意不要漏掉。

5. 角色卡怎么下载

下载时注意看介绍，看看角色卡是在一楼，还是需要主楼链接跳转，最近为了防止卡片贩子，很多是通过回复获取下载链接的，注意认真看主楼说明；

第一次下载时，不需要考虑替换【世界书】和【正则】，但要注意楼内有没有【快速回复】（也就是 QR）需要下载，【快速回复】暂时没法像正则和世界书一样绑定进角色卡，所以需要你手动下（如何导入见 06 插件 — 快速回复部分）。

  • 如果是在类脑/旅程，注意角色卡正确的后缀是 png 或者 json，别认错卡和单纯的美图。
  • 在网页浏览时直接点开图片放大右键保存的不是正确的角色卡！！！需要通过这个在浏览器中打开（新标签页），再来保存 png 格式的角色卡。
05 用户设置 / 背景 可跳过

这一部分主要是一些自定义的界面和使用设置。自己摸索也非常简单。不影响基础游玩，可以先跳了，有机会再看。我就介绍一下我常用的几个功能。

常用设置一览

1. 导入和保存主题

2. 头像样式可以在方圆中选择（我喜欢矩形）

3. 聊天风格也是一种界面设置，气泡会更类似聊天窗口

4. 主题颜色可以点击色块修改

5. 字体比例是倍率，按阅读习惯更改即可，0.9 是缩小，1.1 是放大

6. 其他选项按需求勾选，如果页面太窄，就调节页面宽度

7. CSS 处是更高级的界面修改，但本质可以通过复制粘贴拼凑——小白也能实现修改字体/UI/代码框样式。

8. 图示两项可以简单理解成，当你改自己的名字时，需不需要一键替换过往聊天记录里你的名字。勾选就是替换，不勾选就是不替换。

9. 背景可以自由导入图片，通过锁，可以将背景和当前的对话窗口绑定。

06 插件基础

插件可以给酒馆提供更丰富的功能。我就介绍几个基础又必备的。主要是有时候在酒馆会下到很多东西，你得知道你下的东西要放哪里用。

在右上方可以安装新插件，输入下载地址后，直接 install for all。

正则

正则一般分为两种，一种是和预设绑定的，一种是和角色卡绑定的。

和预设绑定的，一般是为了整理格式、隐藏思维链，在全局模式下使用。

和角色绑定的正则，一般是为了在角色卡中增添一些功能，在局部模式中使用。

快速回复（QR）

Quick Reply，一般简称 QR，用于快速回复、推动剧情，或者召唤手机界面等情况都能使用。

可以直接导入 QR 组，也可以导入单条 QR。摸不准下载的是什么就两个位置都导一下，有一个会失败。

酒馆登录 UI 美化 可跳过

去主题区下载就可以。按教程替换文件，自定义背景，注意背景图的格式。

07 插件推荐（手动添加）

给了 GitHub 地址的可以去给佬们点个赞，也多去原帖看看原版使用教程吧。

安装插件在【三个箱子】（拓展）区：

1. 酒馆助手 / 前端助手（必备）

玩酒馆必装的插件。基本会飘在类脑/旅程的【酒馆插件】区首页。楼主提供了有魔法和无魔法两个版本的 git url，按你的网络情况选用。安装好后，会出现在扩展列表中，不需要再管。

麻瓜源：https://gitlab.com/novi028/JS-Slash-Runner

魔法源：https://github.com/N0VI028/JS-Slash-Runner.git

2. 酒馆助手 - 脚本库（重要）

现在酒馆助手添加了脚本库功能，导入时要添加为全局，内置库里有一部分脚本就挺好用，剩下的要去帖子里找。

3. 提示词模板（重要）

https://github.com/zonde306/ST-Prompt-Template

4. 消息滚动至开头

地址在 作品集 — 白化蓝染的酒馆。其实在用户设置里关闭自动滚动也行。用这个主要是，AI 消息发完后，消息顶部对齐到页面顶部，更强迫症友好。

5. 数据库（重要，二选一）

建议和记忆插件二选一，两个都开 token 反而会炸，用来管理记忆。

1. 原贴地址：【黑科技】【革命性忆插件】【SQLite与原生双语法】【自定义美化】全自动总结 —— 让AI为你实时维护世界观与角色数据！

在原贴首楼，可以跳转资源导航及最新资源。我们需要的是【本体 + 前端 + 3 正则】，本体和正则直接通过直达快车下载，前端需要去内容导航处下载。

2. 下载资源 - 前端：在 原帖 首楼跳转内容导航，在内容导航找到推荐区的前端帖，在前端贴首楼跳转最新版本前端并下载。

3. 下载资源 - 本体 + 正则：在 原帖 首楼跳转直达快车，下载本体 + 正则。

4. 安装：【本体】和【前端】导入（酒馆助手 - 脚本），【正则】导入（正则），并且都别忘了启用。

5. 配置 API：这一部分看官方教程，酒馆数据库新手教程（主要是第二步，部署 API（必须）这部分）。

6. 记忆表格（重要，二选一）

同样在类脑/旅程的【酒馆插件】区 【记忆表格 V2.0.0】，进贴后有麻瓜和魔法两版。具体使用可以看帖内的介绍。
