酒馆搭建教程——VPS云服务器

以腾讯云 + 1panel 面板为例，从零搭建云端酒馆

📄 原始教程来自飞书云文档 →
建议使用电脑浏览器完成搭建，或者平板浏览器开 PC 模式，确保屏幕大点好操作。搭建完后，玩的时候、做简单管理的时候，用手机浏览器即可。
01 选择服务商

国内服务器推荐腾讯云（新用户 4核4G 一年 78，秒杀价 38）

+ 优势国内访问响应快，便宜，好用。
− 劣势不能直连 Gemini、Claude，需要配置 clash 或直接选择公益站。

国外服务器的优点在于可以直连外网（Gemini、Claude），但如果服务器 IP 不优质，依旧需要配置代理。

1. 腾讯云：cloud.tencent.com，4核4G 3M 新用户 79 一年，秒杀 38 一年。建议配合公益站使用，或者搭配 腾讯云主机移除监控程序 教程。如果配置 clash（mihomo）记得手动换个端口，企鹅发现搞代理会封服务器。

2. 阿里云：aliyun.com，2核2G 99 一年，学生用户第一年免费。

3. 百度云：cloud.baidu.com，2核2G 1M（带宽较低），秒杀价一年 19.9 元，新用户一年 50 元左右。

4. 华为云：huaweicloud.com，目前好像没活动。

5. 亚马逊云：第一年/半年免费。

02 腾讯云——购买与安装

1. 在【限时秒杀】区域里找 38 元的服务器，或者 79 元的轻量服务器，4核4G 3M。建议选离得近的服务器。

2. 购买完后可能会自动安装 OpenCloud，可以手动重装系统（切换系统镜像）。

3. 选择系统镜像，并设置服务器密码，Ubuntu 和 Debian 都可以，Ubuntu 体积大，Debian 更稳定更小巧。密码设置得复杂些，保管好，最后确认即可。

03 管理实例（服务器）

1. 如何找到我的服务器

进入腾讯云官网 console.cloud.tencent.com 后，点击【轻量应用服务器】就能找到自己的服务器。

2. 这里就是管理服务器的地方

服务器的【名字】还叫什么是正常的~ 被打码的位置是公网IP，要记好。公网IP相当于你的互联网门牌号，想从本地随时访问云服务器，都需要认准门牌号。

找到那三个点，可以进行一些基础操作（重启/重装），现在点击进入【管理实例】。

这里能修改服务器名字。【用量与监控】可以用来看流量、系统盘空间、运行内存，一般感觉服务器卡了就是内存占用太高，可以手动重启一下。

接下来确认系统镜像是否是 Ubuntu/Debian，两个都可以一样用~ 注意这个【管理防火墙规则】，一会就得用上。

04 管理防火墙规则

在右下角菜单中，配置出入站规则（有的云会叫安全组规则），在这里可以一键放行几个基础端口。

其实等安装好 1panel 面板后，可以在那边放行端口，但如果具体使用中出现问题，可以回这里检查一下，手动开启需要的端口。

腾讯云可以一键放行的端口列表如下：

如果你使用的云没有一键放行，可以在添加规则处手动放行相应端口，主要就是 22，80，443。腾讯云这个手动添加还智能，常用的端口都给放在可选列表了。

04 手动放行端口

在配置出入站规则中，还可以在添加规则处手动放行指定端口，待会装可视化面板就能用上。

安装新软件时，通常会提示需要放行哪些端口。但很多时候的访问失败都和端口放行失败有关，所以每次安装新软件都别忘了回这里检查。

手动添加时：端口协议选自定义 TCP，并填需要放行的端口；描述处写备注提醒自己。

1panel 面板或宝塔面板要用的端口是在首次安装时随机生成的，一定要填你的系统指定的端口。

05 远程登录

远程登录界面，总之在哪个界面都能找到「登录」。

任意登录方式即可，腾讯云能免密登录，就直接用第一种【免密连接】；第二种登录方式要用的是 root 账户的密码，在购买服务时就设置了，如果忘了，找找重置密码的地方~

06 在终端安装 1panel 面板

1. 登录成功后，正式进入云服务器的【终端】，是一个粘贴【命令】的地方。会显示 root@[主机名]:~#

2. 右键粘贴如下内容并回车，手动安装面板（这是一整行）：

curl -sSL https://resource.fit2cloud.com/1panel/package/quick_start.sh -o quick_start.sh && bash quick_start.sh

3. 出现安装确认后，语言选项用数字选中再回车，其他的回车直接确认。如果问你需不需要镜像加速可以输入 y 后回车（但不用镜像站也行）。

4. 面板安装完成，保存信息

出现如下界面则 1panel 面板安装完成，保存好如下所有登录信息，下次登录就看不到了。1panel 端口、账号密码都是随机生成的。在 VPS 终端里，出现 root@[主机名]:~# 就能确定是指令完成了，没出现就多等等。

面板地址的构成：保存好外部地址，加收藏夹叫【1panel登录页】，地址由【公网IP + 1panel面板端口 + 安全入口】构成。

登录成功后会自动跳转，再加一次收藏夹叫【1panel控制页】，地址由【公网IP + 1panel面板端口】构成。如果未登录状态直接访问控制页，会出现如下提示（不会自动跳转登录页）：

5. 检查端口放行

手动放行端口。可以提前把酒馆端口 8000 一并放行，之后安装新服务也需要回这里检查安全组规则。（之后一般在 1panel 里做放行就够了，只见过少数个例还需要在这里重新手动放行）

一个用于面板的登录（在上一步中随机生成），一个默认为 8000 用于酒馆，8484 用于从本地浏览器控制 clwedR/本地使用云端 clwedR，7897 和 9097 用于 mihomo 的代理（国外服务器不需要），但腾讯云可能会监视你的端口，所以建议从 mihomo 安装开始就自定义端口。

6. 登录面板 / 连接终端

在浏览器中打开外部地址，用 1panel 账号密码登录。接下来在这个「可视化界面」里操作就行了。在左侧的【系统】-【终端】，填入服务器 root 账号的密码，记住认证点击【连接测试】后，才能【保存并连接】。

这一步完成后，剩余的大部分操作都可以在 1panel 面板中完成了，但【重装系统/备份系统/检查端口放行】还是需要回到云服务器的控制台。

≈ 1panel 面板上的两种安装酒馆方式

1. Docker 安装版 —— 服务器地址直接在国外，或者服务器在国内但 Gemini 轮询和 ClwedR 准备搭在抱脸上，也就是服务器不安装代理，这两种情况可以选这个。

+ 优势环境更稳定
− 劣势国内服务器开启 mihomo 代理时，docker 版酒馆会残废，问题很多

2. 原生安装版本

+ 优势可以使用魔法
− 劣势运行环境出错了不好修

这两个版本的酒馆可以同时存在，但不能同时运行！数据除非手动搬运 data，否则也不共通。当然也可以在 config.yaml 里修改 listen 的端口，一个用 8000，一个用 8001 就可以同时运行了。

如果这两种都出现问题，去试试老教程也行：【搭建】VPS云酒馆 (宝塔面板) 作为备用。换管理面板最好直接重装系统（切换系统镜像）。

我用原生酒馆是因为 25 年时，装 mihomo 代理开 TUN 会影响 docker 酒馆访问和 clwedR 的速度，现在不清楚有没有解决。如果不需要 mihomo 搞网络代理，建议直接 docker 酒馆，看日志更方便。原生酒馆也只要多装一个小插件也一样看。但是非常建议用腾讯云时搞个代理，不然下东西太慢了，github 连不上。

07 安装原生酒馆（二选一）

1. 安装 Node 和 npm 环境

在面板左侧打开【终端】：

# 1. 更新软件包列表 sudo apt update # 2. 安装 curl（如果未安装） sudo apt install -y curl # 3. 添加 Node.js 仓库 # 4. 安装 Node.js 和 npm # 建议手动访问 https://deb.nodesource.com/，可以复制安装最新的 curl -fsSL https://deb.nodesource.com/setup_22.x | sudo bash - sudo apt-get install -y nodejs # 5. 验证安装 node -v # 应输出 v18.x.x npm -v # 应输出 9.x.x

可以一行一行复制回车，也可以一起复制回车；使用鼠标右键粘贴，避免 CTRL+V；# 行的内容不会被执行，可以不粘贴；如果用的图片复制，最好把内容发给任意 AI 过一遍，检查有无错误。

成功执行后，后 2-3 行显示 node 和 npm 的版本，最后一行会是 root@[主机名]:~

https://deb.nodesource.com/，可以复制 3-4 需要的指令，而且能安装最新版本

如果安装失败，可能和网络代理有关系，去 mihomo 里切直连，关闭 TUN，然后一行行输入，观察在哪里报错，把错误日志发给 AI 问。

2. 安装酒馆

面板左侧进入终端：

右键粘贴如下内容并回车，手动安装酒馆：

git clone https://github.com/SillyTavern/SillyTavern.git

如果总是失败，用下方这条：

git clone https://ghfast.top/https://github.com/SillyTavern/SillyTavern.git

完整显示以下内容，则完成酒馆的下载：

启动酒馆

第一次安装也不能漏掉这一步。后续如果更新酒馆，需要在守护进程处手动关闭，再次用如下指令启动酒馆：

cd /root/SillyTavern && bash start.sh

在终端继续输入指令，等待执行完成（会比较慢）

出现图示内容则安装基本完成（此时还无法完全顺利访问酒馆，关掉这个终端页面，酒馆也会同时关闭）

3. 允许酒馆后台运行

安装进程管理模块

在左侧终端中，输入命令安装 supervisor：

sudo apt-get install supervisor

在左侧菜单的工具箱中，选择进程守护标签页，并点击初始化：

守护酒馆的进程

【工具箱-进程守护-创建守护进程】按图填写。或者检查你的实际运行目录，可能会因为版本不同实际目录是 /root/SillyTavern/SillyTavern。刚才运行的 start.sh 先会检查环境，然后唤醒 node server.js，所以我们只需要守护后者。

管理酒馆：开/关/看日志

可以在此处重启/关闭/查看日志。在设置中【项目日志】处，可以查看日志，截图报错。（无论求助谁，带上截图得到的回复都会更有针对性哦）

4. 快速命令（可跳过）

在终端旁边即可创建，使用时可以在终端下方一键发送：

# 检查8000端口是否被占用（已经有一个酒馆在运行）查询后会显示 sudo ss -tulnp | grep ':8000' # 关闭进程 sudo kill -9 # 将 替换为实际查到的进程 ID（例如 1234） # 检查环境并启动酒馆（一般第一次启动用，start.sh在检查完环境后会启动node server.js） /root/SillyTavern/start.sh # 直接启动酒馆 cd /root/SillyTavern && node server.js
07 安装 Docker 酒馆（二选一）

1. 在面板左侧打开【终端】，右键粘贴如下内容并回车：

cd /opt/1panel/apps && git clone https://github.com/SillyTavern/SillyTavern.git

cd /opt/1panel/apps && git clone https://ghfast.top/https://github.com/SillyTavern/SillyTavern.git

2. 完整显示以下内容，则完成酒馆的下载：

3. 回到左侧「网站-运行环境」，新建 Node 项目，等待成功启动。第一次启动失败可以删除项目重试/换不同镜像源，总是失败就换原生安装吧。

4. 进入「SillyTavern」文件夹，刷新页面。

如果现在进日志查看，会提示酒馆已启动。127.0.0.1 是本地 IP，云酒馆不在本地，没法通过这个访问，还需要进一步设置。

08 修改基础设置

1. 编辑「config.yaml」文件并保存

这个文件在 /root/SillyTavern 目录下，可以在左侧【文件】处一层层找到，也可在【工具箱-进程守护】处点击直达。

找到以下条目，逐一修改为图示内容，并保存。双引号需要为英文引号，引号内的才是账号密码，可以用默认的不修改。这个账号是基础账号验证模式中要用到的基础账号，仅在待会儿要用一次。

2. 回到【工具箱-进程守护】，重启酒馆，让刚改的设置生效

3. 搞清楚你的云酒馆地址

酒馆地址由两部分构成：【云服务器IP:8000】。你现在访问 1panel 面板使用的就是【云服务器IP:1panel指定端口】。云服务器IP可以在服务器管理处看，相当于你的互联网门牌号。

4. 在浏览器中输入你的酒馆地址，登录即可

用户名密码可自定义，如果使用默认的基础账号，则用户名：user，密码：password。

127.0.0.1 是本机IP，无法用于访问云酒馆！只适用于电脑访问搭建在本电脑上的酒馆。

5. 成功登录后，为管理员账户设置密码

按前面所示修改「config.yaml」文件后，就可以用不同身份登录酒馆。在管理员面板右边的【登出】处，可以退出当前账号。默认管理员账户登录账号是 default-user 不可更改，没有密码。点击小钥匙为管理员账户设置账号密码后，就可以算正式启动了多用户模式。

6. 添加新用户

如果你手上有旧酒馆的 data 文件，只要设置好管理员账户的密码，再将 data 文件夹覆盖云端 data 文件夹，旧的用户账户就全回来了。这一步主要图个简单好记的用户名，也可以在这里设立给朋友用的账户。句柄强制为小写，是用户的登录账号。

7. 安装插件

安装插件时最好登录管理员账户，选择为所有用户添加【install for all】。

推荐三个必备的：

酒馆助手（前端助手）
（麻瓜）https://gitlab.com/novi028/JS-Slash-Runner
（魔法）https://github.com/N0VI028/JS-Slash-Runner.git

记忆表格
国内源：https://gitee.com/muyoou/st-memory-enhancement
国外源：https://github.com/muyoou/st-memory-enhancement

消息提示
https://github.com/Mooooooon/silly-tavern-reminder

8. 编辑「config.yaml」文件，关闭基础验证模式

再次回到「config.yaml」文件，将相关条目修改为 false，就可以关闭基础验证模式。保存文件，并回到【进程守护】页面，重启酒馆。重新在浏览器中输入酒馆地址，此时账号密码可以被浏览器记忆。

09 同步旧酒馆内容

如果你在其它地方安装过酒馆，可以直接覆盖云端「config.yaml」文件和「data」文件夹，但插件仍要手动安装。

1. 云酒馆转云酒馆

原本的数据打压缩包上传后，解压进新酒馆的 data 目录即可。压缩包后缀【.tar.gz】可以直接在云服务器中解压。

2. PC 酒馆转云酒馆

在面板的终端里，用如下内容检查你的服务器能支持哪种格式，并将本地文件打包成对应格式。

# 检查ZIP支持 which unzip && echo "已安装ZIP支持" || echo "未安装ZIP支持" # 检查RAR支持 which unrar && echo "已安装RAR支持" || echo "未安装RAR支持"

返回结果如图：（一般都是支持 .zip，不支持 .rar）

要是都不支持，就在终端处手动安装：

# 安装ZIP支持 sudo apt-get install unzip -y # 安装RAR支持（需确保已启用non-free源） sudo apt-get install non-free-repositories sudo apt-get update sudo apt-get install unrar -y
10 原本好好的面板突然无法登录

1. 先检查一下网址对不对，是不是完整的外部地址。

2. 在云服务器控制台处，远程登录到终端，用指令查看 1panel 运行状态，如有问题可以重启服务或服务器。

3. 如果日志看面板运行完全正常，可能问题出在浏览器上，清空缓存或换个浏览器试试；同时注意不要在浏览器里同时打开太多同一个服务器的 1panel 面板控制界面。

# 查看1panel运行状态： systemctl status 1panel # 如果状态显示 inactive（未运行）或 failed（启动失败），尝试重启服务： systemctl restart 1panel # 查看面板的登录账户名及密码 cat /opt/1panel/logs/install.log | grep password
11 云酒馆更新失败

更新指令是 git pull。

如果更新失败，可能是 node 版本过时了需要更新到最新，再用指令更新。

更新成功后如果无法在【进程守护】处重启酒馆，是因为需要先用指令手动重启一遍酒馆完成一个前置工作，即在终端输入 cd /root/SillyTavern && bash start.sh，手动第一次启动后关闭终端，再回到【进程守护】处重启酒馆即可。
