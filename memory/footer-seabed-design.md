# 首页底部海底祈愿区

## 目标

首页底部不是单独展示一整屏插画，而是作为功能卡片之后的紧凑收尾。它需要继续顶部的手绘水彩和纸张质感，并承载吟唱、互动贝壳与召唤小鱼。

## 页面结构

```text
最后一张功能卡片
  ↓ 轻微重叠
海底祈愿区（约 360–480px）
  ├─ 吟唱文字
  ├─ 可互动贝壳
  ├─ 临时游动的小鱼
  └─ 稀疏沙地、贝壳、珍珠和少量珊瑚
```

## 底图

- 与顶部头图使用同一比例和纸张纹理体系。
- 实际网页展示高度控制在约 `360–480px`。
- 装饰元素集中在底边，不使用左右两侧密集包围。
- 右下角保留一小簇稀疏珊瑚。
- 其他位置只保留少量沙粒、海草或珍珠。
- 中央偏左预留干净沙地，用于放置可互动贝壳。
- 动画贝壳不能画死在底图中。

## 吟唱

保留旧版文案：

> 执掌划水的摸鱼之神啊，请赐予我 ——  
> 永远有年假的假象，永远不加班的自由，永远不必来的周一。

“摸鱼之神”使用主题重点色并可点击。文字整体采用浅灰色，保持克制。

## 贝壳动画

采用三张完整状态帧，而不是拆分上下壳后做机械旋转：

- `shell-closed.png`：闭合状态。
- `shell-half.png`：半开过渡状态。
- `shell-open.png`：完全打开状态。
- `shell-pearl.png`：独立珍珠。

三张贝壳图共享 `520 × 440` 透明画布、相同水平中心和底部锚点。状态按闭合、半开、打开切换，关闭时倒序播放；帧间使用约 `120ms` 的透明度过渡。这样可以保留手绘透视和阴影，又不会发生 CSS 分层旋转常见的铰链僵硬与图层穿帮。

## 小鱼交互

- 点击“摸鱼之神”或贝壳时打开贝壳并召唤 `2–3` 条鱼。
- 小鱼仅在祈愿区内活动，向左右或斜上方游出。
- 使用简单线稿 Canvas 绘制，颜色跟随日夜主题。
- 数量有上限，离开区域后自动销毁。
- 无鱼、无贝壳动画时停止动画循环。
- 不恢复旧版全屏常驻鱼群和心形粒子。

## 维护约束

- 底图、贝壳状态和交互逻辑彼此独立。
- 替换底图不应要求重新制作贝壳。
- 所有贝壳素材必须使用相同画布尺寸、中心和铰链锚点。
- `prefers-reduced-motion` 下取消逐帧过程，直接切换最终状态。

## 实现位置

- 页面结构：`index.html`
- 底部样式：`styles/home.css`
- 贝壳与小鱼逻辑：`scripts/seabed.js`
- 海底底图：`concepts/watercolor-seabed-footer-v2.png`
- 贝壳状态：`concepts/shell/`

底部交互单独放在 `scripts/seabed.js`，不继续增加 `scripts/home.js` 的职责。

## 2026-06-16 交互素材调整

- 贝壳默认保持闭合，鼠标或键盘焦点进入时依次切换为半开、全开，离开后反向闭合。
- 点击贝壳或“摸鱼之神”只负责祈愿与召唤小鱼，不再把贝壳当作开关。
- 珍珠仅在完全打开时出现，并向上收进下半贝壳的视觉中心。
- 小鱼改用 `concepts/fish/fish-1.png` 至 `fish-6.png` 的透明水彩贴图。
- 小鱼风格采用简化叶形身体、淡墨尾鳍和少量内部笔触，不使用写实鳞片、立体眼睛或 Canvas 线框鱼。
- 每次召唤的小鱼节点在游出动画结束后自动移除，最多同时保留十条。

## 2026-06-16 祈愿计数规则

- 贝壳本体负责吸引注意：闭合状态有非常轻的呼吸和气泡，悬停或键盘焦点进入时打开。
- 珍珠负责召唤：贝壳打开后点击珍珠，才从贝壳附近游出一条小鱼。
- “摸鱼之神”文字也可召唤一条小鱼，用作吟唱入口。
- 每次召唤只增加一条鱼，保持克制，不做常规鱼群爆发。
- 累计到第 7 条鱼时，开始出现“摸鱼之神”的回应文案；第 8 条以后循环展示短句彩蛋。
- 贝壳本体点击只打开贝壳，移动端会短暂打开后自动合上，不直接召唤小鱼。

## 2026-06-17 贝壳与常驻鱼群

- 贝壳从右侧改到左侧沙地比例位置，作为和底图绑定的场景元素，而不是随着页面宽度随意漂移的 UI 按钮。
- 贝壳半开状态时珍珠先淡入，完全打开后珍珠放大并上移到下壳内部，避免突然闪现。
- 底部小鱼从“一次性扩散消失”改为常驻游动：初始有两条鱼，点击珍珠或“摸鱼之神”时最多补到七条。
- 小鱼使用水彩贴图 DOM 元素，位置由 `scripts/seabed.js` 的动画循环更新。
- 鼠标在海底区域移动时，小鱼会根据距离缓慢避开，参考旧版首页的逃逸逻辑，但保留当前水彩画风。
- 海底底图降低不透明度，并叠加顶部与底部的半透明融合层，让它更像融入页面背景，而不是一张边界明显的图片。
